# Reglas estables del Repositorio Seismic Descent

## Modificación de código existente
**REGLA DE ORO:** Bajo NINGÚN CONCEPTO debes modificar un archivo de código (`.py`) que ya exista y funcione.
- Todas las iteraciones y mejoras algorítmicas deben realizarse creando NUEVOS archivos.
- Es preferible y totalmente aceptable duplicar código entre archivos si esto evita tocar implementaciones pasadas que ya son estables y sirven de referencia.
- Las tareas de limpieza, refactorización y unificación de código se realizarán posteriormente y solo bajo instrucción explícita del usuario.
